| `cfacshr` | Cumulative share adjustment factor |

Key columns:

| Column | Description |
|---|---|
| `permno` | Permanent security identifier |
| `namedt` / `nameendt` | Date range this name record is valid |

| `exchcd` | Exchange code (`1`=NYSE, `2`=AMEX, `3`=NASDAQ) |

| `comnam` | Company name |

## 5. S&P 500 Historical Membership

### The problem

The ideal source is CRSP's `msp500list` table in the `crsp_a_indexes` schema. However, many university WRDS subscriptions do not include `crsp_a_indexes` (you'll see a "permission denied for schema crsp_a_indexes" error if this is the case).

### The solution

The download script uses a maintained open-source dataset from the [fja05680/sp500](https://github.com/fja05680/sp500) GitHub repository. This dataset provides point-in-time snapshots of S&P 500 constituents since 1996, originally sourced from Andreas Clenow's "Trading Evolved" and kept up to date by the maintainer using Wikipedia and manual research.

The process:

1. **Download the CSV** from GitHub. Each row contains a date and a comma-separated list of all S&P 500 tickers on that date (2,595 snapshots from 1996-01-02 to 2019-01-11).
2. **Convert snapshots to membership intervals.** Walk through snapshots chronologically: when a ticker first appears, that's its start date; when it disappears, that's its end date.
3. **Strip date disambiguation suffixes.** The CSV appends `-YYYYMM` to tickers that were reused by different companies (e.g., `CCB-199602` vs `CCB-200305`). These suffixes are stripped after building intervals but before matching to CRSP.
4. **Map tickers to CRSP permnos** using the CRSP name history file, with variant matching to handle convention differences between Yahoo/Wikipedia tickers and CRSP tickers.

### Ticker-to-PERMNO matching

CRSP and Yahoo/Wikipedia use different ticker conventions. The script tries multiple variants for each ticker:

| Source ticker | Variants tried | CRSP convention |
|---|---|---|
| `BRK.B` | `BRK.B`, `BRKB`, `BRK-B`, `BRK B`, `BRK` | `BRK` (class distinguished by permno) |
| `BF.B` | `BF.B`, `BFB`, `BF-B`, `BF B`, `BF` | `BF` |
| `AAMRQ` | `AAMRQ`, `AAMR` | `AAMR` (pre-bankruptcy ticker) |

### Actual results

- **1,127** membership intervals built from 2,595 snapshots
- **1,076** intervals successfully mapped to CRSP permnos (**1,040 unique permnos**)
- **51** intervals unmatched — mostly bankruptcy tickers (`AAMRQ`, `ABKFQ`) and very old/obscure names
- **504** current members (no end date)

### Known limitations

- The GitHub dataset ends at 2019-01-11. Stocks added to the S&P 500 after this date will not be flagged as `is_sp500 = True` in the panel. For our in-sample period (2005-2019), this is not an issue. For the holdout period (2020-2025), the universe will be frozen at its January 2019 composition — this is a known limitation to document in the final report.
- Universe count averages ~445 instead of ~500 due to the 51 unmatched tickers. The sanity checks confirm this does not meaningfully bias the pipeline (tracking error 189 bp, correlation 0.995 vs market return).
- If you obtain `crsp_a_indexes` access in the future, replace the `download_sp500_membership` function with a direct query to `crsp.msp500list` for higher precision.

### Point-in-time membership function

```python
# src/data/universe.py

import pandas as pd
from config import DATA_RAW

def load_sp500_membership() -> pd.DataFrame:
    """Load and clean S&P 500 membership table."""
    df = pd.read_parquet(DATA_RAW / "crsp_sp500_membership.parquet")
    df['ending'] = df['ending'].fillna(pd.Timestamp('2099-12-31'))
    return df

def get_sp500_members(date: pd.Timestamp, membership: pd.DataFrame) -> set:
    """
    Return the set of permnos that were in the S&P 500 on a given date.
    Point-in-time: a stock is a member if start <= date <= ending.
    """
    mask = (membership['start'] <= date) & (membership['ending'] >= date)
    return set(membership.loc[mask, 'permno'])
```

## 6. Delisting Returns

Delisting returns must be incorporated into the daily return series. This is handled automatically by `clean.py`, but the logic is important to understand.

The standard approach (Shumway 1997, Beaver, McNichols & Price 2007):

1. If a stock has a delisting return (`dlret`) on its delisting date, **compound** it with the regular return on that date: `adjusted_ret = (1 + ret) * (1 + dlret) - 1`
2. If the delisting date has no regular return (stock stopped trading before the formal delisting), **use the delisting return alone**: `adjusted_ret = dlret`
3. If `dlret` is missing for a **performance-related delisting** (`dlstcd` 500-599, which includes bankruptcies), **assume `dlret = -0.30`** (Shumway 1997 convention). Without this, bankrupt stocks silently exit with an implicit 0% return.
4. For non-performance delistings with missing `dlret` (mergers, exchange switches), no return is imputed — these are typically benign events.

### Actual results

- **6,673** returns adjusted (all compounded with existing daily returns)
- **164** performance delistings had the Shumway -30% imputation applied

> **Why this matters for momentum:** The short leg of a momentum strategy is full of losers. Losers are the most likely to go bankrupt. Bankruptcy delistings have returns of -30% to -100%. If you drop these stocks instead of incorporating their delisting returns, the short leg looks much better than it actually was, inflating your backtest Sharpe by a meaningful amount.

## 7. Data Cleaning and Assembly

The complete cleaning pipeline is in `src/data/clean.py`. Run it after the download step:

```bash
python -m src.data.clean
```

The pipeline performs these steps in order:

1. **Load raw data** from `data/raw/` Parquet files.
2. **Filter to common stocks on major exchanges** using point-in-time name records (`shrcd` in [10, 11], `exchcd` in [1, 2, 3]). A stock that moved from NYSE to OTC mid-sample is included only for its NYSE days. Actual result: 41.3M → **22.7M rows** (45% removed).
3. **Fix negative prices** — CRSP stores the negative of the bid-ask midpoint when the closing price is missing. Take `abs(prc)`. Actual: **551,338** negative prices fixed.
4. **Compute market cap** — `prc * shrout / 1000` (in millions of dollars).
5. **Merge delisting returns** into the daily return series (Section 6).
6. **Add S&P 500 membership flag** — a boolean `is_sp500` column indicating whether each permno was a member on each date (point-in-time). Actual: **2.1M rows** flagged (9.4% of panel).
7. **Sort, deduplicate, and save** to `data/processed/daily_panel.parquet`.

| `date` | datetime | Trading date |

| `vol` | float | Trading volume (shares) |

## Appendix A: Common Gotchas

1. **Always run scripts from the project root** using module syntax: `python -m src.data.download_crsp`, not `python src/data/download_crsp.py`. This ensures `config.py` is importable.

2. **CRSP returns are already split- and dividend-adjusted.** The `ret` column in `dsf` is a total return. Do not manually adjust. But `prc` is the raw closing price, not adjusted — use `ret` for return calculations.

3. **Negative prices in CRSP.** When the closing price is missing, CRSP stores the negative of the bid-ask midpoint. Always take `abs(prc)`.

4. **PERMNO vs PERMCO vs CUSIP vs ticker.** Use `permno` as your primary identifier. It is stable across ticker changes, name changes, and structural changes. Ticker and CUSIP are not stable.

5. **WRDS query timeouts.** If the daily stock file query times out, download in date chunks (e.g., 5 years at a time) and concatenate locally.

6. **GitHub S&P 500 data has date suffixes on tickers.** The fja05680/sp500 CSV uses `-YYYYMM` suffixes to disambiguate tickers reused by different companies (e.g., `CCB-199602`). The download script strips these automatically before matching to CRSP. If the GitHub repo changes its format, this parsing may need updating.

7. **CRSP ticker conventions differ from Yahoo/Wikipedia.** CRSP stores share classes under the base ticker (`BRK` for both Class A and B), while Yahoo uses `BRK.B`. The download script handles this with variant matching. See Section 5 for details.

## Appendix B: Known Limitations

1. **Universe coverage: ~90% of S&P 500.** 51 out of 1,127 membership intervals could not be mapped to CRSP permnos. These are predominantly bankruptcy tickers and obscure historical names. The impact on momentum signal validation is minimal.

2. **Membership data ends January 2019.** The GitHub dataset's latest snapshot is 2019-01-11. For the holdout period (2020-2025), the universe is frozen at its January 2019 composition. This is acceptable for pipeline validation but would need to be addressed for live trading.

3. **Tracking error of 189 bp vs market.** Higher than the ideal 50 bp due to the ~50-stock shortfall in universe coverage. The 0.995 correlation confirms this is a scale issue (missing stocks), not a data quality issue (wrong returns).
